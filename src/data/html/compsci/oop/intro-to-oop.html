An object can be a list or string etc... However there is such a thing called a class which can be thought of as a template for a group of related items. For example Ducks.

If an object reference is a local variable then it will be stored in the call stack, If it is part of a object then it will be stored in the heap NOT the stack. Not all object references are stored in the stack. Some could be stored in the heap. The object itself will always be stored in the heap.


Time for a quackers DUCK example:

We could have the behaviour of the duck which would be methods of functions:

  • Eat
  • Sleep
  • CommitCrime

For a ducks state we could have properties or attributes or variables:

  • Floatability
  • Crimes
  • Hunger

IMPORTANT

  • Encapsulation
    1. Combining properties and methods together into an object.
    2. Being able to restrict access to an object's state/behaviour.
  • A class is like a blueprint or template for objects. It exists in the code.
  • An object is an instance of a class. It exists in memory (the heap).
  • Instantiation is the process of creating an object from the class.
