Runtime Errors

  • Can be known as Exceptions
  • Takes many forms such as OutOfRangeException or StackOverflowException

These will only occur when the program is run. Very sneaky beaky. You cannot cause these errors if you have outstanding syntax errors.


Catching Exceptions

An exception is also known as a runtime error. To catch an exception is to let the running program recognise when the error occurs. If an exception is left uncaught, the program will crash or cause a debugger to initialize.

If a program is running in prod mode when it crashes it will give a crash report.


Why do we want to catch errors?

  1. To make sure that the program doesnâ€™t close unexpectedly and loose all of your data.
  2. So that we can find where the error is.
  3. Allow for loops when invalid data is provided.

    }

Get a valid number input

Condition: number >= 0 and number <= 100

                    
    static void Main() {
        int number = 0;
        Console.WriteLine("Input a number");

        do {
            try {
                number = int.Parse(Console.ReadLine());
            } catch(FormatException) {
                Console.WriteLine("Invalid Number Please Try Again: ")
            }
        } while (number <= 0);
    }
