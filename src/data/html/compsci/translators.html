Translators

A translator is a piece of system software that is used to convert between a high level language and code that a computer can execute.

NEVER write that a computer can understand something (they can’t) If you must then say it can parse it.

ALWAYS say that it is something that they can execute


ALWAYS say that it is something that they can execute

Assembler

  • Is used with assembly language.
  • It translates between assembly language and machine code.
  • Effectively just a find and replace.

Compiler

  • A compiler is used to take a high level language and convert it into a low level one.
  • It takes the HLL and translates it down into machine code (native code).
  • In a compiler it does not go directly to the hardware it stops half way at the machine code. The same as an assembler.

Interpreter

  • This take the code directly and executes it on the system hardware.
  • It skips the machine code stage.

Native Code

This is the code where it can no longer be run on other hardware. Once it reaches this stage and or you have written machine code it will only work on specific hardware.

Source Code

This is the code that a programmer write themselves. It is the source of the program.

Object Code / Machine Code

Can be used interchangeable however they are different things.


Compiler

Will compiled all of the code at once meaning that it can be slow since it needs to go through the whole thing all at once. However once the code has been compiled it will be able to be run as many times has needed on the same machine hardware.

Key Points

  • Does not execute the program.
  • Need a startup time to complete the compilation.
  • Creates an object filed that can be run without the original source code and programming environment.
  • Object file will be much faster than interpreted.
  • Can hide original source code from the end user.

Interpreter

With this the lines of the code are executed one by one. This means that all you need is the source code and you will be able to run it. This does not convert into object code however it also does not execute the raw high level language. This is far slower from a compiled language this is because it has to run through each line one at a time.

  • Does not produce object code
  • Interpret and execute code per line
  • Can start straight away
  • Only need to interpret the line being run
  • Need time to interpret as the program runs every time the program runs
  • Need original source code and programming environment every time

Byte code

It is an intermediate language that sits between the high level language and the machine code. This is the byte code. Its not quite as high level as a traditional HLL, however it is also not as low as a LLL. Byte code will be much faster to execute since it is created and compiled by a compiler. However when it comes to time to run it it will be interpreted then executed.

C# is an example of a language that uses byte code. It does not go straight to machine code. Instead it translates down to bytecode. When the bytecode is run it is typically interpreted.

It is a good compromise between compilation and interpretation.

It has the massive advantage of being easy to interpret and very easy to move around since it is independent and non native to a particular processor. Portableness.

Note: It doesn’t strictly always doing this sometimes it does some funky stuff precompiling and interpreting as it goes on the fly. It can remember this precompiled code if its needed. Don’t need to know this kinda cool tho.

IMPORTANT: The way that a language is translated is NOT down to the language itself. It is not a feature of the language.
