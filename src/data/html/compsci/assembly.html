Assembly

Assembly language is the term that is given to programs that are written in close to machine code. This is a very low level language. If this comes up in an exam then you will need to hand write.

Machine code is what the processor directly uses to run a program. Assembly is very easily interpreted as machine code meaning that it can be very heavily optimised. Very memory efficient and time efficient.

Binary: 0001 0001 0000 1111

Assembly: ADD R1, #15

The process to get from one to the other is called the decode / encode step. In order to make assembly you would have to use an assembler. The instruction set is a set of binary codes which tell a processor to do a certain thing. It is machine specific.


Opcode

  • These are short abbreviations for what an instruction should do.
  • They are called mnemonics
  • They are typically between 1 and 4 letters
  • EG. ADD, AND, B, BNE, HALT

Assembly language itself consists of the OPCODE and its OPERAND(S). Overall it is a much more human-friendly version of machine code meaning that it is actually possible for a human to ode rather than it being just 1’s and 0’s.

There is no ambiguity with assembly, what you write is what happens in a very literally way. This is because all the assembler has to do in order to convert it into machine code is a find and replace. Very speedy.


Assembly Instructions

  • They do very little per instruction.
  • They can move data around
  • They can perform simple logical and arithmetic operation on data in registers.
  • Simple branching, this means that you can jump to a specific instruction in the code (allows looping)
  • Other instructions such as HALT

Example of Some

                    
    LDR R0, 157
    Start:
    ADD R0, R0, #1
    B start

Addessing Modes

This tells an instruction where to get a piece of data. There are three main types of modes.

  1. Immediate - The data is hard-coded into the program.
    • EG. ADD R0, R1, #7
  2. Direct (register) - The data is stored in a general purpose register.
    • EG. ADD R0, R1, R2
  3. Direct (memory) - The value in the instruction reversed to the address in main system memory where the data is fetched from
    • EG. LDR R0, 5
# denotes immediate addressing. R denotes Direct register if none apply then it is a direct memory address.
